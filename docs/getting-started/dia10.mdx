O último dia é dedicado à apresentação do trabalho desenvolvido, reflexão sobre o processo de aprendizagem e encerramento da sprint. É o momento de demonstrar as funcionalidades implementadas, partilhar conhecimentos entre grupos e realizar a retrospectiva final do projeto para identificar melhorias contínuas.

## Foco
- Sprint review e demonstração final

## Self-learning
- Revisão de sprint
- Retrospectiva
- Ligações aprendidas
- Comparação entre soluções dos grupos

## Tarefas de desenvolvimento
- Criar testes unitários simples
- Definir cenários de aceitação
- Executar testes manuais
- Corrigir bugs
