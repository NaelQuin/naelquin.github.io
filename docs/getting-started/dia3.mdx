O terceiro dia foca-se no **GitLab** e no controlo de versões. Estabelecer um fluxo de trabalho sólido com Git é crucial para o desenvolvimento colaborativo, permitindo a gestão de código, rastreamento de tarefas (Issues) e a integração contínua do trabalho de todos os membros.

## Foco
- GitLab e fluxo de desenvolvimento
## Self-learning
- Pull
- Push
- Stage
- Commit
- Branches 
- Merge Request
- Main branch
- Dev branch
- Feature branch

## Tarefas de desenvolvimento
- Configurar fluxo de branches
- Criar primeiras issues
- Abrir feature branches
- Realizar primeiros commits
- Integração na dev branch

- Fluxo GitLab em funcionamento
