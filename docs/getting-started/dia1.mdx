O primeiro dia é dedicado ao **Kickoff** do projeto, marcando o início oficial da jornada. É o momento fundamental para alinhar a visão do produto, formar as equipas, compreender o problema a resolver e preparar o ambiente de trabalho e ferramentas para as semanas seguintes.

## Foco
- Kickoff
- Visão geral
- Setup Self-learning

## Self-learning
- O que é um sistema Web
- Diferença entre frontend, backend e MVC
- Scrum
- Git e GitLab

## Tarefas de desenvolvimento
- Criar repositório
- Configurar ambiente
- Estruturar projeto
- Definir grupos
- Criar board no GitLab

## Entregável
- Ambiente preparado e projeto criado
