## 1. Guia Prático - Fase 1 (Configuração e Padronização)
Este tutorial serve como guia passo a passo para que os estagiários preparem os documentos físicos, configurem o ambiente de desenvolvimento e preparem os arquivos digitais para a etapa de automação com o Gemini.
### 📋 Passo 1: Padronização das Tags (Tarefa 1.1 e 1.2)
Antes de tocar em qualquer linha de código, o fluxo de dados começa na organização física. O padrão adotado para rastreabilidade universal deve seguir a seguinte convenção de nomenclatura:
 * **Exemplo:** INQ-2026-0001, INQ-2026-0002, etc.
 * **Regra de Aplicação:** O autocolante com o ID (ou código de barras gerado) deve ser colado obrigatoriamente no **canto superior direito** da primeira página do inquérito físico. Isto garante que a região da imagem seja sempre previsível caso decidamos recortá-la programaticamente no futuro.
### 💻 Passo 2: Configuração do Ambiente Python (Tarefa 1.4)
Os estagiários devem isolar as dependências do projeto para evitar conflitos de bibliotecas. Vamos utilizar o venv (nativo do Python) ou o conda.
#### Opção A: Utilizando Python venv (Recomendado para sistemas leves)
Abra o terminal na pasta raiz do projeto e execute os seguintes comandos:
```bash
# 1. Criar o ambiente virtual chamado 'env_inqueritos'
python -m venv env_inqueritos

# 2. Ativar o ambiente virtual
# No Windows (PowerShell):
.\env_inqueritos\Scripts\Activate.ps1
# No Windows (Prompt de Comando clássico):
.\env_inqueritos\Scripts\activate.bat
# No Linux ou macOS:
source env_inqueritos/bin/activate

# 3. Atualizar o gerenciador de pacotes (pip)
python -m pip install --upgrade pip

```
#### Opção B: Utilizando Anaconda/Miniconda
```bash
# 1. Criar o ambiente com a versão estável do Python 3.11
conda create --name env_inqueritos python=3.11 -y

# 2. Ativar o ambiente
conda activate env_inqueritos

```
### 📦 Passo 3: Instalação das Bibliotecas Iniciais
Com o ambiente ativado (o nome do ambiente deve aparecer entre parênteses no início da linha do terminal), execute o comando abaixo para instalar as ferramentas necessárias para a manipulação de arquivos, imagens, variáveis de ambiente e a integração do LangChain com o Gemini:
```bash
pip install opencv-python pillow python-dotenv pandas openpyxl langchain-core langchain-google-genai

```
### 📷 Passo 4: Validação e Renomeação Automática de Imagens (Tarefa 1.3)
Após digitalizarem os inquéritos no scanner (em resolução mínima de **300 DPI**, salvos em formato .png ou .jpg), os estagiários podem se deparar com arquivos nomeados aleatoriamente pelo software do scanner (ex: scan_001.png, photo_2.jpg).
Abaixo está o script em Python que os estagiários devem utilizar para validar se as imagens estão na resolução correta e renomeá-las em lote usando uma lista correspondente de IDs físicos que eles etiquetaram.
Crie um arquivo chamado validar_e_renomear.py:
```python
import os
import cv2
from PIL import Image

def verificar_e_renomear_lote(pasta_origem, lista_tags):
    """
    Verifica os requisitos mínimos da imagem digitalizada e renomeia o arquivo 
    para o ID correspondente da etiqueta física.
    """
    # Lista e ordena os arquivos de imagem na pasta
    arquivos = sorted([f for f in os.listdir(pasta_origem) if f.lower().endswith(('.png', '.jpg', '.jpeg'))])
    
    if len(arquivos) != len(lista_tags):
        print(f"❌ Erro de correspondência! Encontradas {len(arquivos)} imagens, mas fornecidas {len(lista_tags)} tags.")
        return

    print("🔍 Iniciando validação e renomeação...")
    
    for arquivo, nova_tag in zip(arquivos, lista_tags):
        caminho_antigo = os.path.join(pasta_origem, arquivo)
        
        # 1. Validar a resolução usando OpenCV
        img = cv2.imread(caminho_antigo)
        altura, largura, _ = img.shape
        
        # Validação básica: Um documento A4 escaneado a 300 DPI possui aprox. 2480x3508 pixels
        if largura < 2000 or altura < 2000:
            print(f"⚠️ Aviso: A imagem {arquivo} pode estar abaixo de 300 DPI ({largura}x{altura}). Verifique a qualidade.")
        
        # 2. Definir a extensão e o novo caminho
        extensao = os.path.splitext(arquivo)[1].lower()
        novo_nome = f"{nova_tag}{extensao}"
        caminho_novo = os.path.join(pasta_origem, novo_nome)
        
        # 3. Renomear o arquivo com a Tag Universal
        os.rename(caminho_antigo, caminho_novo)
        print(f"✅ {arquivo} -> {novo_nome} | Dimensões: {largura}x{altura}px")

if __name__ == "__main__":
    # Simulação da pasta de scans do projeto
    PASTA_IMAGENS = "./scans_temporarios"
    
    # Criando pasta de teste caso não exista
    if not os.path.exists(PASTA_IMAGENS):
        os.makedirs(PASTA_IMAGENS)
        print(f"Pasta '{PASTA_IMAGENS}' criada. Insira algumas imagens de teste nela antes de rodar o script.")
    else:
        # Exemplo de mapeamento sequencial gerado a partir dos autocolantes colados
        # Em cenários reais, essa lista pode ser lida de um arquivo CSV básico.
        tags_físicas = ["INQ-2026-0001", "INQ-2026-0002", "INQ-2026-0003"]
        
        verificar_e_renomear_lote(PASTA_IMAGENS, tags_físicas)

```
#### Para executar o script:
```bash
python validar_e_renomear.py

```
### 🔒 Passo 5: Configuração do Arquivo de Ambiente .env
Como boa prática de segurança que deve ser seguida desde o primeiro dia, os alunos devem criar um arquivo chamado .env na raiz do projeto para armazenar as credenciais do Google AI Studio que serão usadas para alimentar o Gemini nas fases seguintes.
Exemplo do arquivo .env (os alunos devem preencher com a chave real gerada por eles):
```env
# Chave de API para o modelo Gemini da Google (usada nas Fases 2 e 3)
GOOGLE_API_KEY=AIzaSyA1B2C3D4_SUA_CHAVE_GEMINI_REAL_AQUI...

# Configurações do Projeto
PASTA_PRODUCAO_IMAGENS="./docs_digitalizados_validos"
CAMINHO_EXCEL_OUTPUT="./respostas_consolidadas.xlsx"

```
Para garantir que o Python e o ecossistema LangChain conseguem ler corretamente a credencial do Gemini, eles devem validar a configuração criando um arquivo de teste rápido chamado teste_env.py:
```python
import os
from dotenv import load_model, load_dotenv

# Carrega as variáveis do arquivo .env
load_dotenv()

gemini_key = os.getenv("GOOGLE_API_KEY")

if gemini_key and not gemini_key.startswith("AIzaSyA1B2C3D4_SUA_CHAVE"):
    print("✅ Variáveis de ambiente e GOOGLE_API_KEY carregadas com sucesso!")
else:
    print("❌ Falha ao carregar as variáveis. Verifique se o seu arquivo .env contém a GOOGLE_API_KEY válida.")

```
#### Para executar a validação:
```bash
python teste_env.py

```
