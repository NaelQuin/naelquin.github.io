Getting Started

A comprehensive roadmap outlining project goals and the sequence of tasks required for full implementation.

📄️Dia 1 - Kickoff

O primeiro dia é dedicado ao Kickoff do projeto, marcando o início oficial da jornada. É o momento fundamental para alinhar a visão do produto, formar as equipas, compreender o problema a resolver e preparar o ambiente de trabalho e ferramentas para as semanas seguintes.

📄️Dia 2 - Scrum

No segundo dia, mergulhamos nas Metodologias Ágeis, com foco particular no framework Scrum. Compreender os rituais, papéis e a mentalidade iterativa é essencial para garantir uma entrega de valor contínua e uma colaboração eficiente dentro da equipa.

📄️Dia 3 - Gitlab

O terceiro dia foca-se no GitLab e no controlo de versões. Estabelecer um fluxo de trabalho sólido com Git é crucial para o desenvolvimento colaborativo, permitindo a gestão de código, rastreamento de tarefas (Issues) e a integração contínua do trabalho de todos os membros.

📄️Dia 4 - Frontend

O quarto dia foca-se nos fundamentos do Frontend, explorando a biblioteca React. É o ponto de partida para a construção da interface do utilizador, onde aprendemos a criar componentes reutilizáveis, a gerir o estado da aplicação e a lidar com o fluxo de dados.

📄️Dia 5 - Backend

No quinto dia, exploramos o Backend, a "engrenagem" que corre no servidor. Focamo-nos na criação de APIs, lógica de negócio e na forma como o servidor processa pedidos, garantindo que os dados são manipulados e servidos de forma segura e eficiente.

📄️Dia 6 - Base de Dados

O sexto dia é dedicado à Base de Dados. Aprendemos a modelar dados, criar esquemas relacionais e realizar consultas. Uma base de dados bem estruturada é o alicerce de qualquer aplicação robusta, permitindo o armazenamento persistente e a recuperação rápida de informação.

📄️Dia 7 - Full-stack

No sétimo dia, focamo-nos na integração Full-stack. É o momento de ligar o Frontend ao Backend, permitindo que a interface do utilizador comunique com o servidor e a base de dados em tempo real, criando uma experiência de aplicação completa e dinâmica.

📄️Dia 8 - Arquitetura MVC

O oitavo dia foca-se na Arquitetura MVC (Model-View-Controller). Exploramos como organizar o código de forma modular e escalável, separando a lógica de dados, a interface e o controlo, o que facilita a manutenção e a evolução do projeto a longo prazo.

📄️Dia 9 - Testes

No nono dia, abordamos a importância dos Testes e da qualidade de software. Aprendemos a validar o funcionamento do código através de testes unitários e de integração, garantindo que a aplicação se comporta como esperado e prevenindo a introdução de erros futuros.

📄️Dia 10 - Sprint review

O último dia é dedicado à apresentação do trabalho desenvolvido, reflexão sobre o processo de aprendizagem e encerramento da sprint. É o momento de demonstrar as funcionalidades implementadas, partilhar conhecimentos entre grupos e realizar a retrospectiva final do projeto para identificar melhorias contínuas.
